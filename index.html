      Refresh Notes

        How Refresh Works

        The refresh button runs fully in JavaScript. It reads cleaned/accepted_papers.csv, identifies the conference series, fetches configured official CFP pages through browser-accessible proxies, extracts submission dates, keeps multiple cycle deadlines when they exist, and stores the updated cache in your browser. The analytics tabs read the accepted-paper CSV directly and render their charts fully client-side too.

        Folders

        Styling lives in website/css, browser logic lives in website/js, and the deadline cache lives in website/db as a deploy-time seed. The only HTML entry point is this root index.html.

        GitHub Pages Note

        GitHub Pages cannot rewrite files at runtime, so refreshes are saved in browser storage instead of editing files on the server. Use Download Cache JSON if you want to export the refreshed cache and commit it back into website/db/conference_deadlines.json later.
